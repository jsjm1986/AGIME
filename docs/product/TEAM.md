# 团队协作指南

AGIME Team (v2.5) 专为现代分布式团队设计。

### 为什么团队需要 AGIME？
- **资产沉淀**: 将员工的个人经验转化为团队的**标准化数字资产** (Skills/Recipes)。
- **安全协作**: 在不牺牲隐私的前提下，实现知识与能力的无缝流转。
- **效率倍增**: 一人优化工作流，全员即刻受益，复制销冠/资深工程师的最佳实践。

---

## 1. 协作模式架构

### LAN Mode (局域网模式)
**轻量级、零配置、极速。**
- **适用场景**: 办公室 Wi-Fi 环境、家庭网络。
- **原理**: 利用 mDNS 和 P2P 技术，在局域网内自动发现并直连。
- **优势**: 无需部署服务器，数据完全不经过公网，速度最快。

### Cloud Mode (云端模式)
**企业级、跨网络、高可用。**
- **适用场景**: 远程办公团队、跨国公司。
- **原理**: 部署私有 AGIME Server 作为中继和权限管理中心。
- **优势**: 支持复杂的企业级权限管理 (RBAC)，支持公网访问。

## 2. 资源共享体系

在 AGIME 中，所有知识资产都可以被打包、版本化并共享。

| 资源类型 | 说明 | 示例 |
|:---|:---|:---|
| **Skills (技能)** | AI 的新能力 | "Java 代码审查专家", "合同风险提取器" |
| **Recipes (食谱)** | 标准作业流程 (SOP) | "周报生成流程", "每日竞品监控流程" |
| **Extensions (扩展)** | 连接外部工具 | "PostgreSQL 连接器", "Slack 通知插件" |

### 资源权限矩阵

| 共享级别 | 可见范围 | 适用场景 |
|:---|:---|:---|
| **Public** | 团队全员 | 通用工具（如：请假流程） |
| **Team Only** | 特定小组成员 | 项目组内部资料 |
| **Admin Only** | 仅管理员 | 敏感运维脚本 |
| **Owner Only** | 仅自己 | 个人私有调试 |

---

## 3. 成员角色 (RBAC)

| 角色 | 权限 | 职责 |
|:---|:---|:---|
| **Owner** | 高 | 团队拥有者，掌握最高权限，负责付费与账单。 |
| **Admin** | 中 | 系统管理员，负责成员管理、敏感资源审核。 |
| **Member** | 低 | 普通成员，日常使用、分享资源。 |

### 详细权限表

| 操作 | Owner | Admin | Member |
|:---|:---:|:---:|:---:|
| 邀请/移除成员 | 是 | 是 | 否 |
| 修改成员角色 | 是 | 否 | 否 |
| 强制删除资源 | 是 | 是 | 否 |
| 安装共享资源 | 是 | 是 | 是 |

---

## 4. 邀请与加入流程

1. **管理员发起**: 在 Settings -> Team 面板点击 "Invite Member"。
2. **生成链接**: 设置有效期（如 24小时）和最大使用次数。
3. **成员加入**: 新成员在 AGIME 端输入邀请码，自动同步团队资源。

> **安全提示**: 所有邀请链接均经过加密签名，防止伪造。
