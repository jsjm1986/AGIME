## ✨ 核心功能

### 🤖 智能对话
用自然语言交流，像和助理说话一样。支持上下文记忆，理解你的意图。

### 🔄 食谱系统（工作流复用）
把一次成功的对话保存为"食谱"，下次一键重复执行。

```
场景：每月需要生成相同格式的报告
→ 第一次让 AGIME 帮你做，保存为食谱
→ 以后每月点击执行，自动完成
```

### ⏱️ 定时调度
设置任务自动执行，每小时、每天、每周、每月，灵活配置。

### 🧩 扩展系统（MCP 插件）
通过插件无限扩展能力：

| 插件 | 功能 |
|------|------|
| **Developer** | 代码分析、文件编辑、命令执行 |
| **ComputerController** | 自动化操作电脑 |
| **Memory** | 记住你的偏好和习惯 |
| **Playwright** | 自动浏览网页、填表、截图 |

### 🧠 智能记忆
AGIME 会记住你的偏好，越用越懂你。

### ⚡ 四种工作模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 🟢 **自主模式** | AI 自由操作 | 信任的重复任务 |
| 🟡 **智能模式** | 高风险操作需确认 | 日常使用推荐 |
| 🔴 **手动模式** | 每步都需确认 | 敏感操作 |
| ⚪ **聊天模式** | 仅对话，不执行 | 咨询问答 |

## 👥 团队协作系统

AGIME 支持多人协作，不只是“共享聊天记录”，而是共享可复用的 AI 能力和工作成果。

### Team Server 到底解决什么问题

很多团队卡在三件事：

- 经验在个人电脑里，别人拿不到
- 新人加入后重复踩坑，协作成本高
- 敏感能力要共享，但又担心失控

Team Server 的作用，就是把“个人 AI 使用”升级为“团队协作系统”。

### 你可以和团队共享什么

| 资源类型 | 可以做什么 |
|------|----------|
| **Skills（技能）** | 把某类任务经验沉淀成“可直接复用”的能力 |
| **Recipes（食谱）** | 把完整流程保存为模板，后续一键复用 |
| **Extensions（扩展）** | 统一团队可用工具，减少重复配置 |
| **Documents（文档）** | 团队文档统一管理、检索、协作使用 |

### 资源访问控制（简单理解）

| 级别 | 适合场景 |
|------|------|
| `public` | 团队内广泛共享，快速复用 |
| `team_installable` | 可安装到本地使用，但需要授权 |
| `team_online_only` | 仅在线使用，避免资源落地到个人设备 |
| `controlled` | 严格控制与审计要求较高的场景 |

### 团队角色

| 角色 | 说明 |
|------|------|
| `owner` | 团队负责人，拥有最高管理权限 |
| `admin` | 协助管理成员、资源与邀请 |
| `member` | 使用与协作成员，专注业务执行 |

### Server 协作功能（面向业务）

| 功能 | 你会得到什么 |
|------|------|
| **团队与成员管理** | 快速创建团队、邀请成员、按角色分权 |
| **用户分组管理** | 可按部门/项目组组织成员，做更精细的协作分配 |
| **团队设置中心** | 统一配置审核规则、文档分析等团队级策略 |
| **资源共享与安装** | 技能/流程/扩展可一键分发到团队，减少重复配置 |
| **资源审核机制** | 先审核再开放，降低风险资源被误用的概率 |
| **文档协作中心** | 文档上传、分类、检索、版本管理，团队共用上下文 |
| **统一检索** | 在一个入口搜索团队内多种资源，减少“找资料”时间 |
| **同步与批量更新** | 检查更新、批量安装、统一升级，减少版本不一致问题 |
| **Team Agent** | 服务端持续执行任务，不依赖某个成员一直在线 |
| **Mission 多步骤执行** | 将复杂目标拆解为步骤，按策略推进 |
| **Portal 对外发布** | 把团队内容和能力发布给外部访问者 |
| **Portal 访客互动** | 外部访客可聊天、查看绑定资料、进行受控交互 |
| **日志与审计** | 关键动作可追踪，便于复盘与合规 |
| **统计与趋势** | 了解哪些资源最有价值、哪些流程最常用 |
| **AI 资源解读与洞察** | 自动生成资源描述和团队洞察，帮助快速理解与决策 |

### 协作方式

| 模式 | 场景 |
|------|------|
| **本地/LAN** | 同办公室或同内网团队，部署简单、延迟低 |
| **云端** | 跨地区团队，统一接入、随时协作 |

### Team Server（给管理者）

如果你只是使用者，可以跳过这一段。  
如果你负责部署团队服务，请重点关注：

| 能力 | MongoDB | SQLite |
|------|:-------:|:------:|
| 基础团队协作 | ✅ | ✅ |
| 高级能力（Agent / Portal / 审计统计等） | ✅ | ❌ |

建议：团队正式使用优先选择 **MongoDB**。

### 典型协作场景

| 场景 | 团队怎么用 | 结果 |
|------|------|------|
| **新人入组 1 天上手** | 管理员发邀请，新人加入后直接获得团队技能与流程模板 | 从“自己摸索”变成“按团队标准直接开工” |
| **销售周报自动化** | 把“拉数据-分析-出报告”沉淀成 Recipe，团队统一复用 | 周报产出更稳定，减少人工重复劳动 |
| **法务/财务敏感流程** | 资源设为 `team_online_only` 或 `controlled` | 既共享能力，又降低外泄风险 |
| **跨地协作（北京/上海）** | 云端 Team Server 统一管理，成员随时接入 | 经验同步快，跨地协作不断层 |
| **对外知识门户** | 用 Portal 发布项目资料、FAQ、可交互页面 | 对外沟通更标准，减少重复答疑 |
| **AI 代办复杂任务** | 团队 Agent 在服务端执行多步骤任务并持续反馈进度 | 人只做关键决策，执行效率更高 |
| **多人共享同一知识门户** | 对外统一发布 Portal，客户和合作方都看同一版本信息 | 外部沟通口径统一、减少反复解释 |
| **大型团队分组治理** | 按部门/项目建立用户组并分配资源策略 | 权限更清晰，协作边界更明确 |
| **合规敏感团队上线** | 开启注册审批和资源审核机制后再放开协作 | 保证效率同时兼顾安全与治理 |

### 一条完整协作链路（从 0 到 1）

1. Owner 创建团队并配置规则。  
2. Admin 邀请成员，成员加入后看到统一资源。  
3. 团队把高频流程沉淀为 Skills/Recipes。  
4. Team Agent 接手重复任务，成员专注高价值判断。  
5. 关键成果通过 Portal 对外发布。  
6. 管理层通过日志/审计/统计持续优化流程。

## 🤖 Team Agent（MongoDB Only）

团队可在服务端运行专属 Agent，用于持续执行任务，而不是每次人工盯着。

你可以理解为三类能力：

- **实时对话**：和团队 Agent 多轮沟通，逐步完成复杂工作
- **任务执行**：提交任务后后台运行，完成后返回结果
- **Mission 流程**：将复杂目标拆解为步骤，按策略推进执行

## 🏗️ 技术架构

### 本地优先
桌面客户端核心流程本地运行，团队协作通过可选 Team Server 提供集中式能力。

### 服务器版（`agime-team-server`）

- 提供统一团队协作后端能力
- 支持 Web 管理台（部署后可视化管理）
- 支持 MongoDB / SQLite（功能范围不同，见上文）
- 支持多种登录与认证方式
- 内置后台清理与恢复机制，保障长期稳定运行

### 通用模型兼容
支持市面上几乎所有主流 AI 模型，包括：
- **云端模型** - 通过 API 接入各大模型服务商
- **本地模型** - 通过 Ollama 等方案完全离线运行

> 💡 你可以根据任务需求和隐私要求，自由选择最适合的模型。

### MCP 扩展协议
基于 Model Context Protocol 标准，支持丰富的插件生态：
- 🔧 **工具扩展** - 文件操作、命令执行、网页浏览
- 🔗 **服务集成** - 连接各类外部服务和 API
- 🎨 **自定义能力** - 开发专属插件满足个性化需求

### 安全沙箱
所有操作在受控环境中执行，支持多级权限管理，确保系统安全。

### 跨平台支持
原生支持 Windows、macOS、Linux，统一的用户体验。
